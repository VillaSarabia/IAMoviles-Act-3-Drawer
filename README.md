## Prompt
genera el código completo para una aplicación de tienda de cartas Pokémon llamada DeckTCG estructurada en múltiples archivos. La aplicación debe utilizar una paleta de colores basada en una Super Ball con azul vibrante (#2196F3), blanco (#FFFFFF) y gris oscuro (#37474F). Configura el archivo main.dart con un MaterialApp que establezca un ThemeData global donde todos los AppBar tengan fondo azul y texto blanco, definiendo además un mapa de rutas nombradas para cinco pantallas: la principal y cuatro secciones adicionales. Crea un widget personalizado en la carpeta widgets llamado CustomDrawer que implemente un UserAccountsDrawerHeader azul con un avatar circular que cargue una imagen de una Super Ball desde GitHub, mostrando el nombre "DeckTCG", la dirección "Pueblo Paleta #151", un teléfono y un correo de contacto. Este Drawer debe contener cuatro opciones de navegación (ListTile) con iconos de Material Design para "Cartas TCG" (style), "Entrenadores" (groups), "Accesorios" (shopping_bag) y "Promociones" (local_offer), cada uno vinculando a su ruta respectiva. Para la lógica de las páginas, organiza una carpeta llamada LasPaginas que contenga cinco archivos independientes: home_page.dart, cartas_page.dart, entrenadores_page.dart, accesorios_page.dart y promociones_page.dart. Cada una de estas páginas debe incluir un Scaffold con un AppBar azul coherente y un cuerpo centralizado que muestre una imagen de 200x200 píxeles obtenida de los repositorios de arte oficial de PokeAPI en GitHub, seguida de un texto descriptivo personalizado debajo de la imagen que explique el contenido de esa sección utilizando la fuente estándar en color gris oscuro. Asegúrate de que todas las importaciones entre archivos sean correctas y que el código siga las mejores prácticas de Flutter para una arquitectura limpia y escalable.
